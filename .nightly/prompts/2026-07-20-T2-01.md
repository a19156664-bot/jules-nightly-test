# T2-01: state.yml 日次スナップショット機能の追加

## 背景
state.yml は司令塔の唯一の記憶であり単一障害点（R2）。
バックアップや履歴がないため、書き込み失敗や PC 障害で記憶喪失になるリスクがある。
日次リセット時にスナップショットを保存することで、復旧手段を確保する。

## 仕様

### 変更ファイル
- `commander/state_manager.py` — `reset_daily_if_needed` にスナップショット処理を追加
- `tests/test_state_manager.py` — スナップショットのテストを追加

### 機能
`StateManager.reset_daily_if_needed()` の中で、日次リセットを行う **直前** に
現在の state.yml の内容を `commander/logs/state-snapshot-YYYY-MM-DD.yml` にコピーする。

具体的な処理：
1. `reset_daily_if_needed()` が「リセットが必要」と判断した場合（`last_reset_date != today_str`）
2. リセット処理の **前** に、現在の state を `commander/logs/state-snapshot-{last_reset_date}.yml` として保存
   - `last_reset_date` が None の場合はスナップショットをスキップ（初回起動時）
3. ログディレクトリ（`commander/logs/`）が存在しない場合は作成する
4. スナップショットの保存に失敗してもリセット処理は続行する（try-except で囲む）
5. その後、従来通りリセット処理を行う

### スナップショットファイル形式
- パス: `commander/logs/state-snapshot-YYYY-MM-DD.yml`
- 内容: state.yml と同じ YAML 形式（`yaml.dump` で書き出し）
- エンコーディング: UTF-8（`encoding="utf-8"`, `allow_unicode=True`）

### config.py の LOG_DIR を使用
- スナップショットの保存先は `config.LOG_DIR`（`commander/logs/`）を使用する
- `from commander.config import LOG_DIR` で import
- `config.resolve_path` または `Path` で絶対パスに解決

### テスト要件（tests/test_state_manager.py に追加）
1. `test_snapshot_created_on_daily_reset`: 日付が変わった時にスナップショットファイルが作成される
2. `test_snapshot_content_matches_pre_reset_state`: スナップショットの内容がリセット前の状態と一致する
3. `test_snapshot_skipped_when_no_last_reset_date`: `last_reset_date` が None の場合はスナップショットが作成されない
4. `test_snapshot_failure_does_not_block_reset`: スナップショット保存が失敗してもリセットは正常に完了する

テストでは tmp_path を使用し、StateManager のパスを一時ディレクトリに向ける。

## 完了条件
1. `python -m pytest tests/test_state_manager.py` が全件パス（既存テスト含む）
2. 日次リセット時に `commander/logs/state-snapshot-YYYY-MM-DD.yml` が作成される
3. スナップショット保存の失敗がリセット処理をブロックしない
4. config.py への変更がない

## 注意事項
- commander/ ディレクトリ内のファイルのみ変更すること
- state_manager.py は既に `from commander.config import ...` を使用している（#18 で対応済み）ので、必要な定数を追加で import する
- ファイル書き込みは必ず `encoding="utf-8"` を指定
- .gitignore で `commander/logs/` は除外済みなので、スナップショットは git 管理外
