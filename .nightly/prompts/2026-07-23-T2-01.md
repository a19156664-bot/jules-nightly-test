# T2-01: validate_proposal にプロンプト必須セクション検査を追加

## 背景

タスク設計基準では、各プロンプト（`Tx-01.md`）は
**完了条件・変更可能ファイル・変更禁止ファイル**を明示することになっている
（COMMANDER.md「タスク設計基準」）。しかし `validate_proposal.py` は
プロンプトファイルの**存在**は確認するものの、**中身**は検査しない。

必須セクションが欠けたプロンプトは、Jules に渡した際にスコープや完了条件が
曖昧になり手戻りの原因となる。本タスクでは、proposal 内の各プロンプト
Markdown が必須セクション見出しを含むかを機械的に検査する。

※ 本タスクは T1-01 が integration ブランチへマージされた後に着手される
（turn2 は turn1 完了後に実行されるため、T1-01 の変更は取り込み済みの前提でよい）。

## 完了条件

1. `validate_proposal(proposal_dir_str)` に、各タスクの `prompt_file` に対応する
   proposal 内プロンプト（`Tx-01.md`）について、次の3つの見出しが
   すべて含まれることを検証する処理を追加すること:
   - `## 完了条件`
   - `## 変更可能ファイル`
   - `## 変更禁止ファイル`

2. 判定は、ファイルを UTF-8 で読み、各見出し文字列が本文中に
   （行頭の `## ` に続く見出しとして）存在するかで行うこと。
   欠けている見出しごとに errors に1件ずつ追加すること
   （例: `T2-01.md is missing required section: ## 変更禁止ファイル`）。

3. プロンプトファイルが存在しない場合は、既存の「prompt file 欠落」エラーに
   任せ、本セクション検査では追加のエラーを出さないこと
   （ファイルが存在するときのみ中身を検査する）。

4. 既存の検証項目・出力形式・exit code は変更しないこと。新チェックの失格も
   `VALIDATION: FAIL` + `- <理由>` の複数行形式で報告されること。

5. 既存テストを壊さないこと。`tests/test_validate_proposal.py` に以下を追加:
   - 3セクションすべてを含むプロンプトを持つ正常 proposal が PASS すること。
   - `## 変更禁止ファイル` を欠いたプロンプトが FAIL し、理由に該当見出しが
     示されること。
   - 既存の「正常 proposal が PASS」テストが、追加要件でも通るよう
     テスト用プロンプト本文に3セクションを含めて更新すること
     （既存テストの意図を保ったまま本文を補うこと。テスト自体の削除・スキップは禁止）。

## 技術的注意事項

- 実行は `python -m commander.validate_proposal <dir>` 形式。
- 見出しの照合は素朴な部分文字列一致で十分だが、`## 完了条件` が
  `### 完了条件詳細` のような別見出しに誤マッチしないよう、行単位で
  `line.strip() == "## 完了条件"` のように厳密一致を推奨する。
- ファイル読み込みは UTF-8 を明示すること。
- 対応プロンプトのファイル名解決は既存の `prompt_file` 末尾 `Tx-01.md`
  抽出ロジックを再利用すること（重複実装を避ける）。
- 既存モジュールは import・変更しない。本スクリプトは自己完結のまま保つこと。
