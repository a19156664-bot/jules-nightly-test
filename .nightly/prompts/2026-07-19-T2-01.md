# T2-01: ループエンジニアリング ロードマップ文書 (ROADMAP.md) の新規作成

## 背景
このプロジェクトは「ループエンジニアリング」— 設計→実装→レビュー→再設計を
すべて LLM で閉じるシステムを構築しています。プロジェクト全体の見通しを
文書化し、人間の週次確認にも使えるロードマップを作成します。

## 作成するファイル
- `ROADMAP.md`（新規作成、リポジトリルート）

## 仕様

以下のセクションを含む Markdown 文書を作成すること:

### 1. プロジェクト概要
- ループエンジニアリングの目的を3-4行で説明
- 役割分担: 司令塔（Claude Code）= レビュー・タスク設計 / Jules = コード生成 / 人間 = 方針設定

### 2. アーキテクチャ概要
- 主要コンポーネント: 夜間パイプライン（.nightly/）、司令塔（commander/）、WebUI（webui/）
- 各コンポーネントの責務を1-2行で説明
- データフロー: tasks.yml → Jules → PR → CI → auto-merge → turn-switch のサイクル

### 3. 段階移行表
以下の段階を表形式で記述:

| Phase | 名称 | 説明 | 完了条件 | 状態 |
|---|---|---|---|---|
| P0 | 夜間パイプライン | Jules による自動コード生成 | 2夜連続完走 | ✅ |
| P1 | 自動チェーン検証 | start-night → merge → turn-switch の自動連鎖 | チェーン全要素が動作 | ✅ |
| P2 | 司令塔PoC | 手動起動で司令塔が1回転（判断→アクション） | claude -p で1回転成功 | ✅ |
| P3 | 定時起動 | タスクスケジューラで30分ごと自動起動 | 日中に複数回転が安定稼働 | 次の目標 |
| P4 | 24時間稼働 | 安全弁（停止条件・予算）の実地検証 | 24時間無人稼働で問題なし | 未着手 |
| P5 | UI統合 | WebUI で状態確認・設定変更 | HTML UI で state.yml の閲覧・編集が可能 | 未着手 |

### 4. 安全設計
- 5層の破壊防止: ALWAYS_PROTECTED / protected_paths / レビュー6項目 / 停止条件 / 予算制限
- 禁止事項の概要（停止条件の自己緩和禁止、1起動1アクション等）

### 5. 既知の制限事項
- dispatch 自動発火の GITHUB_TOKEN 制限（ワークフロー分離で対応済み・検証中）
- Pro プランの5時間ローリング制限（2段ゲート + 予算設計で対応）
- models.py と実 tasks.yml スキーマの不整合（P5 で対応予定）
