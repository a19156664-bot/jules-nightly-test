# T1-01: build_prompt.py の新規作成（プロンプト組み立て自動化）

## 背景
現在 commander.ps1 が PowerShell の文字列連結でプロンプトを組み立てている（憲法 + state.yml + prompt-suffix.txt）。
この処理を Python に移すことで：
- PowerShell の cp932 / UTF-8 文字化けリスクを排除
- open alert 件数のプロンプト埋め込みを実現
- config.py のパス定数を活用した一元管理

## 仕様

### ファイル配置
- `commander/build_prompt.py` — 本体
- `tests/test_build_prompt.py` — テスト

### 機能
`build_prompt.py` は以下の処理を行い、組み立てたプロンプト文字列を **stdout に UTF-8 で出力** する。

1. `config.COMMANDER_MD`（`.nightly/COMMANDER.MD`）を読み込む（必須。不在なら exit 1）
2. `config.STATE_FILE`（`commander/state.yml`）を読み込み、`---\n## state.yml:\n` + 内容 を追記（不在ならスキップ）
3. `commander/prompt-suffix.txt` を読み込み追記（不在ならスキップ）
4. 末尾に `\n## 補足情報:\n- open alert 件数: {N}\n` を追記
   - alert 件数は引数 `--alert-count N` で受け取る（デフォルト 0）
5. 組み立てた文字列全体を stdout に print する

### CLI インターフェース
```
python commander/build_prompt.py [--alert-count N] [--output FILE]
```
- `--alert-count N`: open alert 件数（デフォルト 0、整数）
- `--output FILE`: 指定時はファイルに書き出す（未指定なら stdout）
- exit 0: 成功 / exit 1: 必須ファイル不在

### 依存
- 標準ライブラリのみ（argparse, pathlib）
- `commander.config` から `COMMANDER_MD`, `STATE_FILE`, `resolve_path` を import

### テスト要件（tests/test_build_prompt.py）
1. 全ファイル存在時に正しい順序で連結される
2. state.yml 不在時にスキップされる
3. prompt-suffix.txt 不在時にスキップされる
4. COMMANDER.MD 不在時に exit code 1
5. `--alert-count 3` で「open alert 件数: 3」が含まれる
6. `--output` 指定時にファイルに書き出される

テストでは実際のリポジトリファイルを使わず、tmp_path に一時ファイルを作成してテストする。
config のパスは monkeypatch で上書きすること。

## 完了条件
1. `python -m pytest tests/test_build_prompt.py` が全件パス
2. `python commander/build_prompt.py --alert-count 0` が正常にプロンプト文字列を出力
3. 外部コマンド・ネットワークへの依存がない
4. 既存ファイル（commander.ps1, config.py 等）への変更がない

## 注意事項
- commander/ ディレクトリに配置すること（.nightly/ は ALWAYS_PROTECTED）
- ファイル読み込みは必ず `encoding="utf-8"` を指定
- 新規ファイルのみ作成し、既存ファイルは一切変更しない
