# T1-01: 司令塔設定モジュール (config.py) の新規作成

## 背景
司令塔システム（commander/）では、各スクリプトがファイルパスや定数を
ハードコードしています。設定を一元管理するモジュールを作成し、
パスや閾値の変更を1箇所で行えるようにします。

## 作成するファイル
- `commander/config.py`（新規作成）
- `tests/test_config.py`（新規作成）

## 仕様

### commander/config.py

以下の設定を定数またはdataclass/dictで定義すること:

```python
# パス設定
STATE_FILE = "commander/state.yml"
COMMANDER_MD = ".nightly/COMMANDER.md"
TASKS_FILE = ".nightly/tasks.yml"
PROMPTS_DIR = ".nightly/prompts"
LOG_DIR = "commander/logs"

# 予算設定
MAX_LLM_CALLS_PER_DAY = 24
MAX_LLM_CALLS_PER_WINDOW = 8
WINDOW_HOURS = 5

# 停止閾値
MAX_CONSECUTIVE_ERRORS = 5

# 司令塔モデル
DEFAULT_MODEL = "claude-opus-4-8"

# タイムゾーン
import datetime
JST = datetime.timezone(datetime.timedelta(hours=9))
```

### 実装上の制約
- 依存は標準ライブラリのみ（PyYAML 不要）
- Windows / Linux 両対応（pathlib 使用）
- すべての定数・関数に docstring を付けること

## テスト
`tests/test_config.py` を作成し、pytest で以下を検証:
1. STATE_FILE 等の定数が文字列であること
2. JST が UTC+9 であること
3. get_repo_root() が Path を返すこと
4. resolve_path() が絶対パスを返すこと
5. ensure_log_dir() がディレクトリを作成すること（tmpdir 使用）
