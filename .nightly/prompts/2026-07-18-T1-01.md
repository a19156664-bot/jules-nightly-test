## 仕様

### state.yml のスキーマ（デフォルト値）
ファイルが存在しない場合、load() は以下のデフォルト構造を返すこと:

```yaml
version: 1
model: "claude-opus-4-8"
phase: "P2"
loop_status: "idle"        # idle / reviewing / dispatching / waiting / stopped
last_action:
  type: "none"             # review / dispatch / alert / none
  timestamp: null
  result: null             # approved / rejected / error / null
  detail: null
current_night: null
turn: null                 # turn1 / turn2 / complete / null
pending_reviews: []        # レビュー待ち PR 番号のリスト
pending_tasks: []          # dispatch 済み・未 PR のタスク ID リスト
error_count: 0             # 連続エラー回数
budget:
  llm_calls_today: 0       # LLM 実呼び出し回数（日次）
  max_llm_calls_per_day: 24
  llm_calls_window: []     # 直近の LLM 呼び出し時刻 (ISO8601 文字列) のリスト
  max_llm_calls_per_window: 8   # 5時間ローリングウィンドウ内の上限
  wakeups_today: 0         # 起動回数（LLM を呼ばなかった回も含む、参考記録）
  last_reset_date: null    # 最後に日次リセットした日付 "YYYY-MM-DD"
stop_reason: null          # 停止理由。null なら稼働可
```

### クラス: StateManager

```python
class StateManager:
    def __init__(self, path: str = "commander/state.yml"): ...

    def load(self) -> dict:
        """state.yml を読み込む。存在しなければデフォルト構造を返す（ファイルは作らない）"""

    def save(self, state: dict) -> None:
        """atomic write（一時ファイルに書いて os.replace）で保存する"""

    def get(self, key: str, default=None):
        """ドット記法でネストした値を取得する。例: get("last_action.type")"""

    def update(self, updates: dict) -> dict:
        """ドット記法キーの dict で部分更新して save し、更新後の state を返す。
        例: update({"loop_status": "reviewing", "last_action.type": "review"})"""

    def record_action(self, action_type: str, result: str, detail: str) -> None:
        """last_action を現在時刻(JST, ISO8601)で更新して save"""

    def record_wakeup(self, note: str = "") -> None:
        """wakeups_today += 1 して save（日次リセット判定込み）"""

    def record_llm_call(self) -> None:
        """llm_calls_today += 1、llm_calls_window に現在時刻を append して save"""

    def can_call_llm(self) -> tuple[bool, str | None]:
        """LLM 呼び出し可否を判定する。
        - llm_calls_window から5時間以上前の時刻を除去した上で判定
        - 日次上限超過 → (False, "daily-budget-exceeded")
        - ウィンドウ上限超過 → (False, "window-budget-exceeded")
        - stop_reason が非 null → (False, f"stopped: {stop_reason}")
        - それ以外 → (True, None)"""

    def should_stop(self) -> tuple[bool, str | None]:
        """恒久停止条件の判定。
        - error_count >= 5 → (True, "consecutive-errors")
        - stop_reason が非 null → (True, stop_reason)
        - それ以外 → (False, None)"""

    def reset_daily_if_needed(self) -> bool:
        """budget.last_reset_date が今日(JST)でなければ
        llm_calls_today / wakeups_today を 0 にリセットして True を返す"""
```

### CLI インターフェース
`python commander/state_manager.py <command>` で以下を実行可能にすること。
出力は PowerShell から parse しやすいよう **1行のみ** とする。

| コマンド | 動作 | 標準出力 |
|---|---|---|
| `--can-call-llm` | can_call_llm() | `True` または `False\|<reason>` |
| `--should-stop` | should_stop() | `False` または `True\|<reason>` |
| `--record-wakeup <note>` | record_wakeup(note) | `OK` |
| `--record-llm-call` | record_llm_call() | `OK` |
| `--get <key>` | get(key) | 値（null は空文字） |
| `--check-turn-due` | 現在時刻(JST)が turn 切替時刻を過ぎているかの簡易判定 | `True` または `False` |

`--check-turn-due` の判定基準: state の `turn` が `"turn1"` かつ現在時刻(JST)が 21:00 以降なら True、それ以外は False（暫定仕様。将来調整する）。

### 実装上の制約
- 依存は標準ライブラリ + PyYAML のみ（PyYAML はリポジトリで既に使用中）
- タイムゾーンは JST（`datetime.timezone(datetime.timedelta(hours=9))`）
- Windows でも動作すること（パス処理は pathlib を使用）
- すべての公開メソッドに docstring を書くこと
