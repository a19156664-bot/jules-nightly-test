# T2-01: commander.ps1 の Stage 0 に turn=complete 時の LLM 呼び出しパスを追加

## 背景

commander.ps1 は30分ごとにタスクスケジューラから起動される司令塔の起動スクリプトである。
現在の Stage 0 では、open PR が0件かつ check-turn-due が False の場合、
即座に "no-work" として終了する。

しかし turn が "complete"（夜間タスクが全完了）の場合、司令塔（LLM）に
判断表の Row 7（翌夜のタスク設計）を実行させる必要がある。
現在の条件分岐ではこのケースが "no-work" に吸収されてしまい、
司令塔が Row 7 に到達できない。

## 完了条件

1. commander.ps1 の Stage 0 において、open PR が0件かつ check-turn-due が
   False であっても、turn が "complete" の場合は Stage 1（LLM 呼び出し）に
   進むように条件分岐を修正すること。

2. turn の取得には既存の state_manager.py CLI を使用すること:
   ```powershell
   $turn = & $Python -m commander.state_manager --get turn
   ```

3. turn が "complete" 以外（null, turn1, turn2 等）の場合は、
   従来通り alert チェック後に "no-work" で終了すること（既存動作を壊さない）。

4. テストとして、修正後の commander.ps1 を以下の手順で検証できること:
   - state.yml の turn を "complete" に設定した状態で commander.ps1 を実行
   - Stage 1（build_prompt → LLM 呼び出し）に到達することをログで確認
   ※ この手動検証手順を PR の説明に記載すること（自動テストは不要）

## 変更可能ファイル

- `commander/commander.ps1`

## 変更禁止ファイル

- `.nightly/` 配下すべて
- `.github/` 配下すべて
- `AGENTS.md`
- `commander/state_manager.py`
- `commander/config.py`
- `commander/build_prompt.py`
- `commander/parse_output.py`

## 技術的注意事項

- commander.ps1 は PowerShell 5.1 で実行される
- state_manager.py の --get は値を stdout に出力する。
  戻り値の比較は `-like "complete*"` で行うこと（末尾に改行が付く場合があるため）
- 変更は Stage 0 の条件分岐部分（25行目付近）のみ。
  Stage 1 以降は変更しないこと
- $Python 変数は既にスクリプト冒頭で定義済み
