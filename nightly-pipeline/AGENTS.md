# AGENTS.md — プロジェクト憲法(全AIエージェント共通ルール)

このリポジトリで作業するすべてのAIエージェント(Jules等)は、個別タスクの指示に加えて本書を厳守すること。個別指示と本書が矛盾する場合は、**より制限の厳しい方**に従うこと。

## 2. 作業の基本原則
- 指示されたタスクの範囲内でのみ作業すること。「ついで」の改善・リフォーマット・リネームを禁止する。
- 指示に曖昧さや矛盾、コードの実態との食い違いを発見した場合、**推測で埋めずに**、その内容をPR説明文の「未確定事項」に明記すること。
- 障害により完了条件を満たせない場合、回避策を発明せず、できた部分で作業を止めて正直に報告すること。**部分的成果+誠実な報告は、つじつま合わせより常に価値が高い。**

## 3. コーディング規約
<!-- ★導入時に書き換える -->
- 既存コードのスタイル・命名規則・ディレクトリ構成に従うこと。新しい流儀を持ち込まない。
- 変更には目的がコードから読み取れない場合のみ、簡潔なコメントを付すこと。
- **新規の依存パッケージ追加は、タスク指示で明示的に許可された場合のみ**。無断追加を禁止する。

## 4. テストの規律(最重要)
- ロジックを変更・追加した場合、対応するテストを**追加**すること。
- **既存テストの削除・スキップ・期待値の変更・リネームを禁止する。** 既存テストが失敗する場合は、テストを直すのではなく、失敗内容をPR説明文に報告すること(自動検収はテストファイルの「追加」以外の変更を全てブロックする)。
- テストは実装の写経ではなく、タスクの完了条件を検証するものであること。

## 5. 禁止事項(フェイルセーフ)
- 以下のパスに触れてはならない: `.github/`、`.nightly/`、`AGENTS.md`
  <!-- ★プロジェクト固有の保護パスは .nightly/tasks.yml の protected_paths にも追記する -->
- 認証情報・APIキー・トークン・パスワードをコード、コメント、ログ、PR説明文のいずれにも書き込まない。
- 本番環境の設定、データベースのマイグレーション、CI/CD設定の変更を行わない(これらは夜間タスクの対象外である)。
- タスク指示に記載のないファイルの大規模な書き換え(一括フォーマット、import順の整理等)を行わない。

## 6. Pull Request の様式(自動検収との契約)
- **PRタイトルは必ずタスクID `[T1-01] ` の形式で始めること。** これがない場合、自動マージの対象外となる。
- PRの向き先(base)は、タスク指示で指定された `integration/nightly-*` ブランチとすること。`main` へのPRを作成しない。
- PR説明文には以下の4項目を必ず含めること:
  1. **完了条件との対応表** — 指示された各完了条件を、どの変更でどう満たしたか
  2. **変更ファイル一覧と各変更の理由**
  3. **テスト実行結果** — 実行コマンドと結果(実行できなかった場合はその理由)
  4. **未確定事項** — 妥協した点、判断に迷った点、人間に確認してほしい点(なければ「なし」と明記)
