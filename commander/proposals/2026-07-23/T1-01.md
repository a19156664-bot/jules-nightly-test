# T1-01: validate_proposal に night とディレクトリ名の整合チェックを追加

## 背景

`commander/validate_proposal.py`（PR #27 で実装）は proposal ディレクトリの
`tasks.yml` を検証するが、**`tasks.yml` 内の `night` 値と proposal ディレクトリ名が
一致しているか**は検証していない。

実際に運用中の `.nightly/tasks.yml` は `night: "2026-07-21"` のまま
2026-07-22 の夜に使われており、night のコピペミスは現実に発生している。
proposal ディレクトリは `commander/proposals/YYYY-MM-DD/` という命名なので、
ディレクトリ名の日付と `night` が食い違っていれば機械的に検出できるはずである。

本タスクでは、この整合チェックを `validate_proposal.py` に追加する。

## 完了条件

1. `validate_proposal(proposal_dir_str)` に、次の検証を追加すること:
   - proposal ディレクトリ名（`Path(proposal_dir_str).name`）が
     `YYYY-MM-DD` 形式である場合、`tasks.yml` の `night` 値と一致すること。
     一致しなければエラーとして errors に追加する
     （例: `night '2026-07-21' does not match proposal directory name '2026-07-23'`）。
   - ディレクトリ名が `YYYY-MM-DD` 形式でない場合（例: 末尾に `.archived` 等が付く）は、
     この整合チェックはスキップし、エラーとしないこと（既存の運用で
     `2026-07-22.archived-obsolete` のようなディレクトリが存在するため）。
   - `night` が欠落している、または形式不正の場合は既存の該当エラーに任せ、
     本チェックで重複エラーを出さないこと（`night` が有効な `YYYY-MM-DD` の
     ときのみ突き合わせる）。

2. 既存の検証項目・出力形式（`VALIDATION: PASS` / `VALIDATION: FAIL` +
   `- <理由>` の複数行出力、exit code 0/1）は一切変更しないこと。
   新チェックの失格も同じ形式で報告されること。

3. 既存テストを壊さないこと。`tests/test_validate_proposal.py` に以下を追加:
   - ディレクトリ名 `2026-07-23` に対し `night: "2026-07-23"` の proposal が
     PASS すること（`tmp_path` 配下に該当名のサブディレクトリを作って検証）。
   - ディレクトリ名 `2026-07-23` に対し `night: "2026-07-21"` の proposal が
     FAIL し、理由に不一致が示されること。
   - ディレクトリ名が `2026-07-23.archived` のように非日付形式の場合は、
     night が異なっていても本チェックでは FAIL しないこと。

## 技術的注意事項

- 実行は `python -m commander.validate_proposal <dir>` 形式（`docs/lessons.md` 知見30）。
- ディレクトリ名の取得は `Path(proposal_dir_str).name` を用いること。末尾の
  スラッシュ有無で結果が変わらないよう注意（必要なら `.rstrip("/\\")` 後に判定）。
- 既存の `night` 形式チェックの正規表現 `^\d{4}-\d{2}-\d{2}$` を再利用してよい。
- テストは既存の組み立て方式（`tmp_path` に tasks.yml と Tx-01.md を配置）に倣うこと。
- 既存モジュールは import・変更しない。本スクリプトは自己完結のまま保つこと。
