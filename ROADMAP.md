# ループエンジニアリング ロードマップ

## 1. プロジェクト概要

本プロジェクトは、設計、実装、レビュー、再設計のソフトウェア開発サイクルをすべてLLMによって自動完結させる「ループエンジニアリング」のシステム構築を目的としています。
各エージェントの役割分担は以下の通りです：
- 司令塔（Claude Code）: レビューおよびタスク設計を担当
- Jules: コード生成を担当
- 人間: 全体の方針設定を担当

## 2. アーキテクチャ概要

主要コンポーネントとその責務は以下の通りです：
- 夜間パイプライン（`.nightly/`）: CI/CDおよび自動処理の実行を担当
- 司令塔（`commander/`）: タスクの判断とエージェントの制御を担当
- WebUI（`webui/`）: システム状態の確認と設定管理を担当

データフローは `tasks.yml → Jules → PR → CI → auto-merge → turn-switch` のサイクルで進行します。

## 3. 段階移行表

| Phase | 名称 | 説明 | 完了条件 | 状態 |
|---|---|---|---|---|
| P0 | 夜間パイプライン | Jules による自動コード生成 | 2夜連続完走 | ✅ |
| P1 | 自動チェーン検証 | start-night → merge → turn-switch の自動連鎖 | チェーン全要素が動作 | ✅ |

| P4 | 24時間稼働 | 安全弁（停止条件・予算）の実地検証 | 24時間無人稼働で問題なし | 未着手 |
| P5 | UI統合 | WebUI で状態確認・設定変更 | HTML UI で state.yml の閲覧・編集が可能 | 未着手 |

## 4. 安全設計

システムには5層の破壊防止策が組み込まれています：
1. ALWAYS_PROTECTED
2. protected_paths
3. レビュー6項目
4. 停止条件
5. 予算制限

禁止事項の概要：
- 停止条件の自己緩和の禁止
- 1起動につき1アクションのみ実行可能

## 5. 既知の制限事項

- dispatch 自動発火の GITHUB_TOKEN 制限（ワークフロー分離で対応済み・検証中）
- Pro プランの5時間ローリング制限（2段ゲート + 予算設計で対応）
- models.py と実 tasks.yml スキーマの不整合（P5 で対応予定）
